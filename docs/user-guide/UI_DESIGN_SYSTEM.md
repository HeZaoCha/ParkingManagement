## 设计原则

### 1. 核心设计理念

本设计系统基于以下核心理念：

-   **用户中心**: 以用户体验为核心，确保界面直观、易用
-   **一致性**: 保持视觉和交互的一致性，降低学习成本
-   **可访问性**: 遵循 WCAG 2.1 AA 级标准，确保所有用户都能使用
-   **性能优先**: 优化加载速度和交互响应
-   **可维护性**: 使用 CSS 变量和模块化设计，便于维护和扩展
-   **简洁高效**: 减少不必要的元素和层级，突出核心功能
-   **安全可靠**: 尊重用户隐私，提供可感知、可控制的安全体验

### 3. 通用友好原则

参考 [华为 HarmonyOS 通用友好设计原则](https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-overview-0000001900422910):

#### 3.1 即点即用、秒开清爽

-   **启动无开屏、无广告，无重复加载**
    -   通过图标、卡片等方式打开应用，无开屏界面
    -   非用户主动操作，应尽量避免出现广告弹框干扰用户
    -   若确有运营需要，建议将广告内容内嵌入页面之中，或使用更加轻量的底部非模态浮窗
    -   进入启动界面时只有一个加载，无重复加载

-   **一键授权、一键登录、一键退出**
    -   首次进入应用，一键完成隐私声明和用户协议的同意与签署
    -   调用统一的授权能力，一键完成账号个人信息授权
    -   通过系统返回手势、关闭等方式，一键关闭应用。不允许二次弹框打断用户行为

#### 3.2 减少界面信息层级

-   **单页面内不超过 3 个主要的信息层级**
    -   人因实验数据表明，大于 3 级的时候，视觉复杂度会显著的变高
    -   减少界面内层级相互嵌套情况
    -   减少界面层级跳转

#### 3.3 高级简约、自然流畅

-   **界面布局简洁，重点突出**
    -   降低信息复杂性：界面信息简洁，重点信息突出，没有多余或不必要的信息

-   **转场自然流畅，操作响应即时**
    -   自然合理：界面转场要清晰表达层级关系，通过动效传递不同的交互层级隐喻，界面过渡变化符合心理预期
    -   精致流畅：画面变化连贯丝滑，无抖动、卡顿、闪跳等现象
    -   及时可控：操作响应无延迟感，操控感舒适高效

### 4. 简洁高级原则

参考 [华为 HarmonyOS 简洁高级设计原则](https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-overview-0000001900424082):

#### 4.1 可视愉悦，界面平衡

通过平衡性、层级性两个维度来确保界面具有平衡协调的信息组织性，达到舒适愉悦的视觉观感。

-   **平衡性**:
    -   元素图形平衡
    -   元素间大小重量平衡
    -   页面视觉平衡
    -   元素排布对齐、文本排版对齐

-   **层级性**:
    -   **深度层级**: 通过色彩明度、阴影、模糊等视觉手段，构造深度线索。界面元素的前景与背景对比清晰，易于识别
    -   **平面层级**: 通过文本字号、色彩饱和度差异、元素大小差异，来构造平面层级。界面中的文字层级对比清晰，色彩重点突出，元素重点突出

#### 4.2 简洁布局，突出重点

-   **突出重要信息或功能，视觉动线符合用户浏览习惯**
    -   界面设计时，应根据内容和操作的重要程度进行合理的视觉元素排布，确保突出界面中最重要的任务或关键操作
    -   避免过多冗余信息干扰，影响用户操作

-   **降低界面信息密度**
    -   过高的信息密度会提升页面的复杂性，加重用户的认知负荷
    -   过低的信息密度又会导致单个页面呈现的信息量过少，降低用户获取信息的效率
    -   需合理设计界面内的信息呈现，确保界面信息密度保持在一个适当水平：
        -   界面信息种类数适中，包括大小、色彩数量等
        -   界面信息数适中，包括组块数量、文字数量等
        -   适当留白，关键操作的交互控件在界面上足够明显，界面内合理的留白，让界面更加清晰明了

-   **避免广告信息干扰用户，弱化广告感**
    -   高感知的内容与用户当前的任务目标不符时，就会产生广告感
    -   过度的无关信息会导致用户感到界面杂乱，不简洁
    -   价值评估维度：
        -   价值适用：提供与任务相关的信息内容，达成用户目标
        -   分心适度：避免视觉设计引起过渡凸显，维护用户注意力聚焦
        -   中断适时：降低广告组件中断用户任务所带来的的用户成本，且时机得当
        -   数量适中：控制整体数量，保持适度平衡，不过多打扰

#### 4.3 精致细节，体现品质

界面从布局、图标、颜色、材质、文字，保持清晰：

-   **界面布局**: 布局平衡，间距一致
-   **文字质量**: 在不同的背景亮度下，文字保持清晰易读
-   **图片质量**: 图片的显示应清晰易识别，减少模糊感、块面感，色彩还原度高
-   **图标质量**: 图标表意清晰；图标线条粗细、风格一致；使用一种图标风格，单个界面最多两种图标风格

#### 4.4 界面一致，易学易用

每个界面相同场景处理方式保持一致，包括布局、颜色、材质、字体。

#### 4.5 多彩特征，多元有序

-   色彩是用户对界面感知的第一印象，通过不同颜色的运用，可以创造出不同的情感和氛围
-   应用可以使用符合品牌特色的特征色作为界面主色，所有界面操作元素统一一种颜色

### 5. 轻量高效原则

参考 [华为 HarmonyOS 轻量高效设计原则](https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-overview-0000001939183197):

#### 5.1 导航清晰，入口明确

-   **导航应该提供清晰的路径**
    -   用户使用的时候，能够知道当前处在界面的什么位置，操作后将会跳转到什么位置
    -   导航简洁清晰，避免复杂设计

-   **次级页面左上角的返回键交互行为统一定义为"返回上一层"**
    -   而非"返回前一个界面"

-   **每个界面中功能的入口和出口必须明确、清晰、稳定**
    -   通过界面元素，帮助用户清晰认知界面层级和当前所在位置

#### 5.2 轻量架构，高效触达

-   **轻量架构**: 应用的信息架构应保持简洁轻量，界面中只提供对用户有核心价值的内容
    -   一级页面通过底部页签对内容进行组织架构
    -   底部页签的数量最多不超过 5 个，最少不少于 2 个

-   **功能高效触达**: 核心功能的关键操作在首屏可发现、易操作
    -   功能操作是否易发现，可以从复杂操作入口是否有提示、入口发现的困难度这几个维度考虑设计

-   **页面层级合理**: 界面跳转关系与转场效果符合层级逻辑，浅层操作减少界面跳转

#### 5.3 直觉交互，符合预期

-   **遵循基础交互与手势**
    -   屏幕边缘左右滑动返回与退出

-   **操作合理性**:
    -   可见性(状态、变化、内容可见)，任何操作都有实时反馈
    -   不要假设用户的推理能力，要设想用户偏好简单易懂的操作
    -   运用同理心来衡量设计是否符合用户心智模型，确保交互控件的行为和意图符合用户认知

-   **内容易理解性**: 界面中的图标、提示信息易于理解，一看就懂

-   **体验连贯性**: 使用过程中不应被弹窗、跳转等打断主流程

#### 5.4 容错防呆，消除歧义

-   **为用户提供简单易学的引导和提示，降低出错概率**

-   **不可避免出现错误时，除了告知操作失败，应该给出解决方案的建议**
    -   如无网络时，引导用户去设置网络
    -   内容获取失败时，引导用户点击屏幕重试

-   **任何需要用户等待的操作，都需要提供暂停或取消的方式**

#### 5.5 友好表达，通俗易懂

-   **使用用户容易理解的自然语言，避免技术术语词汇与生僻的缩写**

-   **运营性质的界面用语，避免诱导性用语引导用户点击**

#### 5.6 及时反馈，缓解等待

-   数据的获取和加载时长应尽可能缩短，以避免用户长时间等待
-   当不可避免地出现需要加载和等待的场景时，需提供及时的反馈以缓解用户的等待焦虑

### 6. 安全可靠原则

参考 [华为 HarmonyOS 安全可靠设计原则](https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-overview-0000001900584166):

#### 6.1 尊重隐私，可管可控

感知到的安全是积极体验的驱动力，应提供给用户一个可感知、可控制、易使用的安全体验。

-   **信息透明**: 采集个人数据时，应清晰、明确地告知用户，并确保告知用户的个人信息将被如何使用

-   **规避风险**: 帮助用户规避风险，准确认识风险以及后果
    -   在获取用户的个人数据时，可以通过调用系统提供的场景化组件来规避不可控的数据获取
    -   使用安全控件（位置、剪贴板、保存）
    -   使用系统选择器（图库、文件、联系人、扫一扫、相机等）

-   **可控制**: 权限授权、账号个人信息授权用户可控制、可反悔

#### 6.2 安全可信，权威可靠

-   提供权威的安全认证体系
-   具备安全认证的应用，具有完善的消费者权益保证服务
-   在搜索结果页、应用市场等场景会有安全认证标签的露出，提升用户对该应用安全可靠的信赖感

### 7. 响应式设计原则

参考 [阿里云 - 现代前端开发中的响应式设计原则与实践](https://developer.aliyun.com/article/1569493):

#### 7.1 弹性网格布局

-   在响应式设计中，弹性网格布局（Flexible Grid Layout）是至关重要的一环
-   通过使用相对单位如百分比或 em，而非固定单位如像素，网页可以根据屏幕尺寸动态调整布局
-   这样做不仅使网页更具适应性，还能有效地适应不同大小和分辨率的设备

#### 7.2 弹性图片与媒体查询

-   除了布局的弹性外，响应式设计还包括了图片和媒体的适应性
-   通过使用响应式图片（Responsive Images）和媒体查询（Media Queries），可以根据设备的特性（如屏幕分辨率、宽度等）加载合适尺寸和分辨率的图片及媒体资源
-   从而提升页面加载速度和用户体验

#### 7.3 流式布局与断点设计

-   **流式布局（Fluid Layout）**: 是响应式设计中常用的布局方式之一，它允许内容根据浏览器窗口大小自动调整
-   **断点设计（Breakpoint Design）**: 则指定了在不同分辨率下网页布局发生变化的临界点，通过合理设置断点，可以有效地优化不同设备上的显示效果

## 交互设计原则

### 1. 人因设计理念

参考华为 HarmonyOS 人因设计理念，让数字世界遵循自然法则，使交互直观本真，实现人与设备的和谐共生。

#### 1.1 差异性

-   深入了解目标设备的特性，包括屏幕尺寸、交互方式、使用场景和用户群体
-   针对性地进行设计，充分利用设备特性

#### 1.2 一致性

-   在考虑设备特性的同时，识别不同设备的共性
-   采用通用设计方法，确保跨设备的一致性
-   降低用户学习成本和开发成本

#### 1.3 灵活性

-   针对硬件能力、交互方式和使用场景相似的设备，调整布局位置、内容宽度和组件数量
-   避免因屏幕方向切换或窗口尺寸变化导致的界面问题

### 2. 交互反馈原则

#### 2.1 即时反馈

-   任何操作都应有实时反馈
-   状态变化、内容变化应可见
-   操作响应无延迟感，操控感舒适高效

#### 2.2 反馈类型

-   **视觉反馈**: 按钮状态变化、加载动画、进度指示
-   **触觉反馈**: 点击震动、操作确认（移动端）
-   **听觉反馈**: 操作音效（可选，需用户可控制）

#### 2.3 等待反馈

-   数据的获取和加载时长应尽可能缩短
-   当不可避免地出现需要加载和等待的场景时，需提供及时的反馈以缓解用户的等待焦虑
-   任何需要用户等待的操作，都需要提供暂停或取消的方式

### 3. 容错与防错原则

#### 3.1 预防错误

-   为用户提供简单易学的引导和提示，降低出错概率
-   通过设计来预防用户可能的错误操作

#### 3.2 错误处理

-   不可避免出现错误时，除了告知操作失败，应该给出解决方案的建议
-   错误信息应清晰、友好，避免技术术语
-   提供明确的恢复路径

#### 3.3 可撤销性

-   用户应能够控制界面的行为，随时撤销或重做操作
-   重要操作应提供确认机制
-   增强用户的掌控感

## 颜色系统

### 1. 颜色架构

颜色系统采用三层架构：

```
主题色 (Theme Color)
    ↓
亮暗模式 (Light/Dark Mode)
    ↓
背景色/文本色/边框色 (Semantic Colors)
```

### 2. 主题色定义

主题色系统支持四类主题，每类主题包含 5 个变体：

#### 2.1 标准主题（Standard Themes）

-   **蓝色主题** (`blue`): 默认主题，专业、可信
-   **紫色主题** (`purple`): 创意、优雅
-   **绿色主题** (`green`): 自然、安全
-   **橙色主题** (`orange`): 活力、温暖
-   **玫瑰主题** (`rose`): 柔和、友好

#### 2.2 浅色调主题（Light Themes）

-   `blue-light`, `green-light`, `pink-light`, `purple-light`, `orange-light`
-   使用更浅的色调，适合明亮环境

#### 2.3 深色调主题（Dark Themes）

-   `blue-dark`, `purple-dark`, `green-dark`, `orange-dark`, `rose-dark`
-   使用更深的色调，提供更强的视觉冲击

#### 2.4 渐变主题（Gradient Themes）

-   `blue-gradient`, `purple-gradient`, `green-gradient`, `orange-gradient`, `rose-gradient`
-   使用颜色范围创建动态渐变效果，提供丰富的视觉体验

### 3. 颜色变量命名规范

#### 3.1 主题色变量

```css
--color-primary-50   /* 最浅色 */
--color-primary-100
--color-primary-200
--color-primary-300
--color-primary-400
--color-primary-500   /* 主色 */
--color-primary-600
--color-primary-700
--color-primary-800
--color-primary-900   /* 最深色 */
```

#### 3.2 语义颜色变量

```css
/* 背景色 */
--bg-primary         /* 主背景 */
--bg-secondary       /* 次要背景 */
--bg-tertiary        /* 第三级背景 */
--bg-card            /* 卡片背景 */
--bg-hover           /* 悬停背景 */

/* 文本色 */
--text-primary       /* 主要文本 */
--text-secondary     /* 次要文本 */
--text-muted         /* 弱化文本 */

/* 边框色 */
--border-color       /* 边框颜色 */

/* 阴影 */
--shadow-color       /* 阴影颜色 */
```

### 4. 颜色应用规则

#### 4.1 背景色应用

-   **页面背景**: 使用 `--bg-secondary`，提供层次感
-   **卡片背景**: 使用 `--bg-card`，确保内容清晰可读
-   **悬停效果**: 使用 `--bg-hover`，提供交互反馈

#### 4.2 文本色应用

-   **标题和重要文本**: 使用 `--text-primary`，确保高对比度
-   **正文和次要文本**: 使用 `--text-secondary`，保持可读性
-   **提示和辅助文本**: 使用 `--text-muted`，降低视觉权重

#### 4.3 主题色应用

-   **主要操作按钮**: 使用 `--color-primary-500` 或 `--color-primary-600`
-   **悬停状态**: 使用 `--color-primary-600` 或 `--color-primary-700`
-   **激活状态**: 使用 `--color-primary-700` 或 `--color-primary-800`
-   **强调元素**: 使用 `--color-primary-400` 或 `--color-primary-500`

## 亮暗模式设计

### 1. 设计原则

#### 1.1 避免纯黑色背景

-   **原因**: 纯黑色（#000000）背景上的纯白色文本会产生过强的视觉对比，导致眼睛疲劳
-   **解决方案**: 使用深灰色（#121212）作为暗模式主背景
-   **参考**: [素马设计指南](https://www.sumaarts.com/share/1409.html) - "避免使用 100%黑色"

#### 1.2 使用深灰色层次

暗模式背景采用深灰色层次系统：

```css
--bg-primary-dark: #121212    /* 主背景 - 最深灰色 */
--bg-secondary-dark: #1e1e1e  /* 次要背景 - 稍浅 */
--bg-tertiary-dark: #2a2a2a   /* 第三级背景 - 中等深灰色 */
--bg-card-dark: #1e1e1e       /* 卡片背景 */
--bg-hover-dark: #2a2a2a      /* 悬停背景 */
```

**设计理由**:

-   提供视觉层次，避免单调
-   减少眼睛疲劳，比纯黑色更舒适
-   符合现代 UI 设计趋势（Material Design, Apple HIG 等）

#### 1.3 避免饱和色

-   **原因**: 完全饱和的颜色在暗模式下难以聚焦，导致眼睛疲劳
-   **解决方案**: 使用柔和的颜色，降低饱和度
-   **参考**: [素马设计指南](https://www.sumaarts.com/share/1409.html) - "避免使用饱和色"

#### 1.4 文本对比度要求

遵循 WCAG 2.1 标准：

-   **AA 级标准**:

    -   正文文本: 对比度至少 **4.5:1**
    -   大文本（18pt+或 14pt+粗体）: 对比度至少 **3:1**

-   **AAA 级标准**（推荐用于重要文本）:
    -   正文文本: 对比度至少 **7:1**
    -   大文本: 对比度至少 **4.5:1**

**当前实现**:

```css
/* 暗模式文本颜色 */
--text-primary: #f1f5f9; /* 与#121212对比度约15.8:1 ✅ */
--text-secondary: #cbd5e1; /* 与#121212对比度约11.2:1 ✅ */
--text-muted: #94a3b8; /* 与#121212对比度约6.8:1 ✅ */
```

### 2. 亮暗模式切换

#### 2.1 实现方式

使用 CSS 类切换：

```html
<html class="light" data-theme="blue">
    <!-- 或 -->
    <html class="dark" data-theme="blue"></html>
</html>
```

#### 2.2 背景色动态应用

背景色根据主题色和亮暗模式动态生成：

```css
html.light {
    --bg-primary: var(--bg-primary-light, #ffffff);
    --bg-secondary: var(--bg-secondary-light, #f8fafc);
    /* ... */
}

html.dark {
    --bg-primary: var(--bg-primary-dark, #121212);
    --bg-secondary: var(--bg-secondary-dark, #1e1e1e);
    /* ... */
}
```

#### 2.3 用户偏好记忆

-   使用 `localStorage` 存储用户选择的亮暗模式
-   支持系统偏好检测（`prefers-color-scheme`）
-   提供手动切换选项

## 主题色系统

### 1. 主题色与背景色的关系

#### 1.1 设计原则

-   **主题色影响背景色**: 不同主题色应产生不同的背景色氛围
-   **亮暗模式独立**: 每个主题色都有对应的亮色和暗色背景变体
-   **层次分明**: 背景色应提供清晰的视觉层次

#### 1.2 背景色生成规则

**亮色模式**:

-   使用主题色的浅色变体（50-200）作为背景
-   主背景: `--bg-primary-light` = 主题色-50
-   次要背景: `--bg-secondary-light` = 主题色-100
-   第三级背景: `--bg-tertiary-light` = 主题色-200

**暗色模式**:

-   统一使用深灰色系统（#121212, #1e1e1e, #2a2a2a）
-   不因主题色变化而改变暗色背景
-   确保暗模式的一致性和舒适性

**设计理由**:

-   亮色模式: 主题色背景提供品牌识别和视觉氛围
-   暗色模式: 深灰色背景确保眼睛舒适，主题色仅用于强调元素

### 2. 渐变主题设计

#### 2.1 渐变实现

渐变主题使用颜色范围创建动态渐变：

```css
background: linear-gradient(
    -45deg,
    var(--color-primary-900),
    /* 最深色 */ var(--color-primary-700),
    /* 深色 */ var(--color-primary-500),
    /* 主色 */ var(--color-primary-300),
    /* 浅色 */ var(--color-primary-100),
    /* 最浅色 */ var(--color-primary-300),
    /* 浅色 */ var(--color-primary-500),
    /* 主色 */ var(--color-primary-700),
    /* 深色 */ var(--color-primary-900) /* 最深色 */
);
background-size: 400% 400%;
animation: gradient 15s ease infinite;
```

#### 2.2 渐变效果

-   使用 9 个颜色节点，覆盖从最深到最浅的完整范围
-   45 度角渐变，提供动态视觉效果
-   15 秒循环动画，缓慢平滑过渡

## 布局与信息架构

### 1. 信息层级设计

#### 1.1 层级限制原则

-   **单页面内不超过 3 个主要的信息层级**
    -   人因实验数据表明，大于 3 级的时候，视觉复杂度会显著的变高
    -   减少界面内层级相互嵌套情况
    -   减少界面层级跳转

#### 1.2 层级表达方式

-   **深度层级**: 通过色彩明度、阴影、模糊等视觉手段，构造深度线索
-   **平面层级**: 通过文本字号、色彩饱和度差异、元素大小差异，来构造平面层级

### 2. 导航设计

#### 2.1 导航清晰原则

-   导航应该提供清晰的路径
-   用户使用的时候，能够知道当前处在界面的什么位置，操作后将会跳转到什么位置
-   导航简洁清晰，避免复杂设计

#### 2.2 返回行为

-   次级页面左上角的返回键交互行为统一定义为"返回上一层"
-   而非"返回前一个界面"

#### 2.3 入口和出口

-   每个界面中功能的入口和出口必须明确、清晰、稳定
-   通过界面元素，帮助用户清晰认知界面层级和当前所在位置

### 3. 信息架构

#### 3.1 轻量架构

-   应用的信息架构应保持简洁轻量，界面中只提供对用户有核心价值的内容
-   一级页面通过底部页签对内容进行组织架构
-   底部页签的数量最多不超过 5 个，最少不少于 2 个

#### 3.2 功能触达

-   核心功能的关键操作在首屏可发现、易操作
-   功能操作是否易发现，可以从复杂操作入口是否有提示、入口发现的困难度这几个维度考虑设计

#### 3.3 页面层级

-   界面跳转关系与转场效果符合层级逻辑
-   浅层操作减少界面跳转

## 响应式设计

### 1. 设计原则

参考 [RWD 响应式网页设计核心原则](https://www.ibest.com.tw/news-detail/responsive-web-design/):

#### 1.1 信息流（The Flow）

-   内容随屏幕尺寸变化自然流动
-   小屏幕时内容占据更多垂直空间

#### 1.2 相对单位（Relative Units）

-   使用百分比、em、rem 等相对单位
-   确保在不同设备和像素密度下正常显示

#### 1.3 断点（Breakpoints）

-   使用标准断点：`sm` (640px), `md` (768px), `lg` (1024px), `xl` (1280px)
-   根据内容决定断点位置，避免过多断点

#### 1.4 最大和最小值（Max and Min Values）

-   使用 `max-width` 和 `min-width` 限制内容尺寸
-   确保在大屏幕和小屏幕上都有合适的显示

### 2. 移动优先策略

-   从小屏幕开始设计，逐步增强到大屏幕
-   确保核心功能在所有设备上可用
-   优化触摸交互体验

## 可访问性

### 1. WCAG 2.1 合规性

#### 1.1 对比度要求

-   **正文文本**: 至少 4.5:1（AA 级）
-   **大文本**: 至少 3:1（AA 级）
-   **重要文本**: 推荐 7:1（AAA 级）

#### 1.2 颜色使用

-   不单独依赖颜色传达信息
-   确保足够的亮度对比度
-   支持色盲用户（使用形状、图标等辅助标识）

#### 1.3 交互元素

-   确保所有交互元素有足够的点击区域（至少 44x44px）
-   提供清晰的焦点指示
-   支持键盘导航

### 2. ARIA 属性

使用适当的 ARIA 属性增强可访问性：

```html
<input
    type="text"
    aria-describedby="hint error"
    aria-invalid="false"
    aria-required="true"
/>
<div id="hint" class="text-xs text-theme-muted">提示信息</div>
<div id="error" role="alert" aria-live="polite" class="hidden">错误信息</div>
```

## 实施指南

### 1. 使用 CSS 变量

所有颜色应通过 CSS 变量使用，避免硬编码：

```css
/* ✅ 正确 */
.button {
    background-color: var(--color-primary-500);
    color: var(--text-primary);
}

/* ❌ 错误 */
.button {
    background-color: #3b82f6; /* 硬编码颜色 */
    color: #0f172a;
}
```

### 2. 使用语义类名

使用 Tailwind CSS 的语义类名：

```html
<!-- ✅ 正确 -->
<div class="bg-theme-secondary text-theme-primary">内容</div>

<!-- ❌ 错误 -->
<div class="bg-gray-100 text-gray-900">内容</div>
```

### 3. 主题切换实现

```javascript
// 切换主题色
function setTheme(themeName) {
    document.documentElement.setAttribute("data-theme", themeName);
    localStorage.setItem("theme", themeName);
}

// 切换亮暗模式
function setMode(mode) {
    document.documentElement.className = mode;
    localStorage.setItem("mode", mode);
}
```

### 4. 设计原则检查清单

在实施新页面或组件时，请检查：

#### 4.1 通用友好原则

-   [ ] 启动无开屏、无广告，无重复加载
-   [ ] 一键授权、一键登录、一键退出
-   [ ] 单页面内不超过 3 个主要的信息层级
-   [ ] 减少界面内层级相互嵌套情况
-   [ ] 减少界面层级跳转
-   [ ] 转场自然流畅，操作响应即时（无抖动、卡顿、闪跳）

#### 4.2 简洁高级原则

-   [ ] 界面布局简洁，重点突出
-   [ ] 降低界面信息密度（信息种类数适中、信息数适中、适当留白）
-   [ ] 避免广告信息干扰用户，弱化广告感
-   [ ] 界面平衡（元素图形平衡、大小重量平衡、页面视觉平衡、对齐）
-   [ ] 层级清晰（深度层级、平面层级）
-   [ ] 精致细节（布局平衡、文字清晰、图片清晰、图标一致）

#### 4.3 轻量高效原则

-   [ ] 导航清晰，入口明确
-   [ ] 轻量架构（只提供核心价值内容）
-   [ ] 核心功能在首屏可发现、易操作
-   [ ] 直觉交互，符合预期（可见性、易理解、连贯性）
-   [ ] 容错防呆，消除歧义（引导提示、错误解决方案、可取消操作）
-   [ ] 友好表达，通俗易懂（自然语言、避免技术术语）
-   [ ] 及时反馈，缓解等待（加载反馈、进度提示）

#### 4.4 安全可靠原则

-   [ ] 信息透明（清晰告知数据采集和使用方式）
-   [ ] 规避风险（使用系统安全控件和选择器）
-   [ ] 可控制（权限授权可控制、可反悔）

#### 4.5 技术实现检查

-   [ ] 所有颜色使用 CSS 变量
-   [ ] 文本对比度符合 WCAG 标准
-   [ ] 暗模式使用深灰色而非纯黑色
-   [ ] 响应式设计在所有断点下正常
-   [ ] 交互元素有适当的 ARIA 属性
-   [ ] 支持键盘导航
-   [ ] 在不同主题色下测试显示效果
